4. Approval gates — what Claude decides vs human

Autonomous (Claude executes)

  • Generate reports, dashboards, USALI P&L
  • Compute KPIs and pace
  • Recommend BAR within ±15% of last published rate
  • Alert on cancellations, no-show spikes, OOO/OOS
  • Propose LOS restrictions
  • Push approved rates to Cloudbeds

Human approval required

  • Any BAR change > ±15% of last rate
  • Group quotes above $X total value (set threshold)
  • New rate plans / channel contracts
  • Wholesale allotment changes
  • Closing dates / overbooking decisions
  • Strategy shifts (new comp set, new segment focus)

6. Risks & weak spots — be honest

Pricing Agent autonomy

If you graduate to autonomous BAR pushing too fast, one bad data feed = revenue disaster.

→ Mitigation: 90 days recommend-only, then guardrails (floor/ceiling, max ±15% per day, max 3 changes per week per room type).

Comp Set scraping

Scraping OTAs is fragile (anti-bot), legally grey, and rates shown ≠ rates booked.

→ Mitigation: subscribe to Lighthouse or OTA Insight. Budget ~€300–600/mo. Or accept manual weekly shop.

Cloudbeds API limits

API rate limits + folio sync delays mean "real-time" reporting is actually 15–30 min lagged.

→ Mitigation: cache aggressively, run heavy queries off-hours, accept lag for non-pricing decisions.

Hallucinated KPIs

If Claude invents a KPI not in 05_KPI_DEFINITIONS.md, your USALI pack goes wrong.

→ Mitigation: hard rule in Team Lead system prompt — refuse any KPI not defined in the file. Fail loud.

Single point of failure

If Claude Code crashes at 08:55, no flash at 09:00. Operations breaks.

→ Mitigation: cron + health check. Fallback to last-known-good flash. SMS alert to human RM.

Human RM atrophy

If the human stops thinking and rubber-stamps Claude's recommendations, you lose strategic judgment.

→ Mitigation: weekly "challenge session" — human must override at least one Claude recommendation with rationale.
