## How this folder is used

Output of every approved-proposal deployment lands here:

- `deploy-doc-frontend-<section>-<tab>-<YYYY-MM-DD>.md` — frontend deploy record (Next.js / Vercel)
- `deploy-doc-schema-<scope>-<YYYY-MM-DD>.md` — Supabase migration record
- `_LOG.md` — this file, the shared ledger
- (optional) screenshots / build artifacts

The input side lives in `~/Desktop/namkhan-bi/proposals approved/`. See `_FLOW.md` there for the contract.

## Status legend

- `prepared` — deploy doc generated by the IA scheduled task; awaits a Claude Code session to actually run it
- `in-progress` — deploy session running
- `shipped` — `.shipped` marker dropped in `proposals approved/`; live URL or migration confirmed
- `failed` — see deploy doc for reason; proposal goes back to in-progress
- `rolled-back` — shipped but reverted; deploy doc has the revert commit / DOWN-block applied
